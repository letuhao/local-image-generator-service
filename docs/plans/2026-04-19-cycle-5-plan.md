- `app/loras/scanner.py` (new) — `LoraMeta` frozen dataclass, `scan_loras(root: Path) -> list[LoraMeta]`
  - Walk recursively via `Path.rglob("*.safetensors")`
  - Compute `name` = relative-to-root path without `.safetensors` suffix, POSIX-form (`/` separator)
  - Check name against `_NAME_RE` → `addressable` flag + `reason`
  - Load sidecar `<stem>.json` if exists; tolerate `json.JSONDecodeError` → log warning, fall through

  - Return list sorted by `name`

### Chunk 3 — workflows.py (ResolvedLoraRef + inject_loras + inject_vpred)

Files:
- `app/registry/workflows.py` (modify):
  - Add `ResolvedLoraRef` frozen dataclass
  - Add `inject_loras(graph, loras, *, model_cfg)` implementing arch §9 algorithm:
    1. Empty list → return
    2. Find `%MODEL_SOURCE%` + `%CLIP_SOURCE%` anchor node ids
    3. Compute `max_id = max(int(k) for k in graph.keys())`
    4. For each lora: create `LoraLoader` node with string key `str(max_id + i)`; inputs = `{lora_name: f"{name}.safetensors", strength_model: weight, strength_clip: weight, model: [prev_model, 0], clip: [prev_clip, 0]}`
    5. After chain built, rewrite downstream consumers of the anchor nodes' `model` and `clip` outputs to point at the final chain node
    6. Do not touch other anchors (KSampler/outputs) — that's worker's job elsewhere
  - Add `inject_vpred(graph, *, model_cfg)` per spec §8.2
